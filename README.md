# Selectel IP Hunter ☁️🎯

Перебор floating IP в Selectel Cloud Platform до получения адреса с нужным префиксом.
Управление полностью через Telegram-бота. В `.env` нужно заполнить только **2 поля**.

Логика на одну попытку:
1. Создать floating IP в проекте Selectel
2. Если IP подходит под цель — оставить и остановить охоту
3. Если нет — удалить IP и попробовать ещё раз

Скорость по умолчанию **30 попыток в минуту** (одна каждые 2 секунды), чтобы
не нагружать API Selectel. Меняется в настройках бота.

## Быстрый старт

### 1. Создать Telegram-бота

1. Написать [@BotFather](https://t.me/BotFather) → `/newbot`
2. Скопировать токен

### 2. Узнать свой User ID

Написать [@userinfobot](https://t.me/userinfobot) → скопировать `Id`

### 3. Заполнить `.env`

```bash
cp .env.example .env
```

```env
TELEGRAM_BOT_TOKEN=7123456789:AAFxxx...
TELEGRAM_USER_ID=123456789
```

### 4. Запустить

```bash
docker compose up --build -d
```

Логи:

```bash
docker compose logs -f
```

## Что нужно от Selectel

В панели Selectel создайте **сервисного пользователя** с правами на проект:

1. **Управление → Пользователи → Сервисные пользователи → Создать**
2. Дайте ему роль `Member` (или выше) на нужном проекте
3. Запомните: **имя пользователя**, **пароль**, **ID аккаунта** (виден в правом верхнем углу панели), **имя проекта**, **регион**

В проекте должен быть **роутер с внешним шлюзом** — иначе Neutron не сможет
выдать floating IP. Если роутера нет, создайте: **Облачная платформа → Сети → Роутеры**.

## Управление через бота

Напишите боту `/start`:

```
☁️ Selectel IP Hunter
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚪ Ожидание
👤 0 акк  ·  ⚡ 30 rpm

[🎯 Запустить охоту]
[👥 Аккаунты]   [⚙️ Настройки]
```

### Добавление аккаунта

«Аккаунты → Добавить аккаунт». Бот спросит по шагам:

1. Название (любое короткое имя для отображения)
2. Имя сервисного пользователя
3. Пароль *(сообщение с паролем будет удалено сразу)*
4. ID аккаунта Selectel
5. Имя проекта
6. Регион (`ru-1`, `ru-2`, `ru-3`, `ru-7`, `ru-8`, `ru-9`)
7. Целевой IP — см. форматы ниже

### Форматы цели

| Формат              | Что значит                          |
|---------------------|-------------------------------------|
| `5.250.1.2`         | Точное совпадение                   |
| `5.250`             | Содержит подстроку `5.250`          |
| `5.*.*.2`           | Wildcard на 4 октета                |
| `5.250, 95.213`     | Любой из перечисленных (через запятую) |

### Настройки

| Параметр           | По умолчанию | Описание                                  |
|-------------------|--------------|-------------------------------------------|
| Попыток/мин       | 30           | Темп перебора на аккаунт                   |
| Пауза при ошибке  | 5 сек        | Backoff после ошибки API                   |
| Пауза Rate Limit  | 15 сек       | Что делать при HTTP 429                    |
| Обновление, сек   | 4            | Период перерисовки карточки охоты          |

> Не задирайте rpm выше 60. Selectel — провайдер с реальными квотами,
> агрессивный перебор может привести к временной блокировке проекта.

  📡 95.213.155.42  🔄 27  ❌ 0
  ⏱ 00:00:54

При успехе — отдельный алерт с ID floating IP, который остаётся
в проекте и доступен для привязки к ресурсу через панель Selectel.

## Структура

```
selectel-toolbox/
├── docker-compose.yml
├── Dockerfile
├── requirements.txt
├── .env.example
└── src/
    ├── main.py            # entry point
    ├── config.py          # переменные окружения
    ├── storage.py         # JSON-хранилище аккаунтов и настроек
    ├── selectel_client.py # Keystone v3 + Neutron API
    ├── worker.py          # цикл перебора IP
    └── bot.py             # Telegram UI (aiogram 3)
```

Все аккаунты и настройки сохраняются в `./data/config.json` (примонтирован
как том Docker), данные переживают перезапуск контейнера.

## Запуск без Docker

```bash
python -m venv .venv
source .venv/bin/activate     # Linux/macOS
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
python -m src.main
```

`config.json` создастся в `./data/config.json` рядом с проектом.
